# Standalone usage of components

Some applications won't build a search user interface from scratch, but they will possible need a new element in their interface.
Therefore, each of the components in the AMSUI library can be used standalone. Just import the modules that you need and you good to go.

## Available Components

Some components are nested, for example, multiple facet components in the facet group, but they can also just used on their own as well.

- [search field](https://dev.redlink.io/amsui/?path=/docs/searchfield-field--default#search-field-component)
- [facet group](https://dev.redlink.io/amsui/?path=/docs/facets-facet-group--multiple-facets#facet-group-component)
  - [facet](https://dev.redlink.io/amsui/?path=/docs/facets-facet--default#facet-component)
- [search result wrapper](https://dev.redlink.io/amsui/?path=/docs/results-wrapper--default#search-result-wrapper-component)
  - [search results container](https://dev.redlink.io/amsui/?path=/docs/results-results--default#results-component)
  - [search result entry](https://dev.redlink.io/amsui/?path=/docs/results-result-entry--default#result-entry-component)
  - [no results container](https://dev.redlink.io/amsui/?path=/docs/results-no-results--default)
  - [search meta information](https://dev.redlink.io/amsui/?path=/docs/results-meta-information--default#search-result-meta-information-component)
    - [did you mean text](https://dev.redlink.io/amsui/?path=/docs/results-result-view-type--default#result-view-type-component)
    - [result view type toggle](https://dev.redlink.io/amsui/?path=/docs/results-result-view-type--default#result-view-type-component)
    - [meta information text](https://dev.redlink.io/amsui/?path=/docs/results-meta-text--default#search-result-meta-text-component)
    - [sort dropdown](https://dev.redlink.io/amsui/?path=/docs/results-sort-dropdown--default#sort-dropdown-component)
- [overlay loader](https://dev.redlink.io/amsui/?path=/docs/overlay-loader--default#overlay-loader-component)

You can easily import all of the above component's module and just the ones that you need.
The components have defaults, so they work either out-of-the-box or just don't show up.
They do not create errors when adding the corresponding selector tag without any attributes or event handlers in your application's templates.

Just go to the component you want add on the left sidebar menu. You will get a quick intro about each component there.
